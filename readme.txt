Lab_2: мій перший коміт

команди для створення та переключення між гілками:
git checkout lab2
 git checkout master

Конфлікт у git виникає, коли під час злиття двох гілок в одному або кількох файлах відбулися зміни, які не можуть бути автоматично об'єднані. Найчастіше це стається, коли різні гілки вносять зміни до одного й того ж рядка або секції коду, і git не може визначити, яку версію обрати.

Коли виникає конфлікт, git позначає його відповідно у файлах, які містять конфліктні зміни. Зазвичай в таких файлах буде виглядати наступно:
<<<<<<< HEAD
Код з поточної гілки
=======
Код з іншої гілки
>>>>>>> branch_name
Для вирішення конфліктів:

Видаліть маркери конфлікту (<<<<<<<, =======, >>>>>>>) і збережіть файл з вирішеним конфліктом.

Після вирішення всіх конфліктів в усіх файлах, ви можете продовжити злиття гілок за допомогою команди git merge або git pull.
